Experiment 2: Does the Metadata Frame Actually Help?
Full Outline
Research Question
Does the historical metadata frame (author, date, nationality, genre, profession) provide genuine signal to models, or is it decorative? And does it matter what kind of question it is?

Hypothesis

Full metadata > no metadata > shuffled metadata in performance (real frames help; wrong frames actively mislead)
The harm from shuffled metadata will be largest for book_context questions (character, topic sentence) because those questions are about a specific book — wrong metadata is contradictory, not just absent
world_context questions (knowledge, inference) will show smaller gaps — the metadata is contextualizing but not load-bearing
Three Conditions

Condition	Description	File
A — Full	Existing benchmark, unchanged	all_benchmark_questions.jsonl
B — No metadata	metadata_frame replaced with a neutral placeholder	benchmark_no_metadata.jsonl
C — Shuffled	metadata_frame randomly reassigned within each frame_type group	benchmark_shuffled_metadata.jsonl
Shuffling within frame_type ensures a cloze question always gets a cloze-style frame (passage context), not a knowledge-style frame — isolating metadata content from metadata format.

What to Measure

Brier score (primary — more sensitive)
MCQ accuracy + skill score
Both broken down by:
frame_type (world_context / book_context / passage_context)
reasoning_type (knowledge, inference, character_modeling, constrained_generation, topic_sentence, phrase_cloze, sentence_cloze)

Model(s) to Run

Qwen2.5-7B-Instruct (already have baseline results for condition A — can re-run for B and C, or use existing results for A)
any other oen source and free model like gpt oss
